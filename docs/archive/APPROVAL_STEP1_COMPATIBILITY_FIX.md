# Approval Step 1 Compatibility Fix

## Issue Found

The JSON generated by Approval Step 1 uses `extension_id` but the framework expected `interface_id`.

## Root Cause

- **Step 1 Hook**: Generates JSON with `"extension_id": "EXT_FIN_004"`
- **Framework**: Was looking for `"interface_id"` field
- **Result**: Framework would use "Unknown" as the interface ID

## Fix Applied

Updated `ReusableTools/testing_framework/orchestration/test_orchestrator.py` to accept both field names:

```python
# Extract metadata
# Support both 'interface_id' (for interfaces) and 'extension_id' (for approvals)
interface_id = test_data.get('interface_id') or test_data.get('extension_id', 'Unknown')
interface_type = test_data.get('interface_type', 'inbound')
scenarios = test_data.get('scenarios', [])
```

Now the framework will:
1. First try to get `interface_id` (for interface testing)
2. If not found, try to get `extension_id` (for approval testing)
3. If neither found, use "Unknown"
