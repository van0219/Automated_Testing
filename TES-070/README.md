# TES-070 Test Results Documents

This folder contains official TES-070 format test results documents for FSM automated testing.

## 📁 Folder Structure

```
TES-070/
├── Sample_Documents/       # Reference TES-070 examples (3 sample documents)
├── Generated_TES070s/      # Automated test outputs (generated by test scripts)
└── extracted_images/       # Analysis artifacts (OCR, image extraction)
```

## 📄 What is TES-070?

TES-070 is the official test results document format that contains:
- Test scenario descriptions
- Step-by-step test procedures
- Expected vs actual results
- Pass/fail status
- Embedded screenshots as evidence
- Work unit IDs and timestamps

## 🎯 Folder Purposes

### Sample_Documents/
Contains reference TES-070 documents from previous testing:
- Use as templates for new test documents
- Reference for formatting and structure
- Examples of proper test evidence documentation

**Current samples:**

### Generated_TES070s/
Stores newly created TES-070 documents from automated testing:
- Output from test script executions
- Generated test evidence and screenshots
- Work-in-progress test documentation
- Final test results ready for delivery

### extracted_images/
Analysis artifacts from TES-070 analyzer tool:

- OCR analysis results
- Image categorization reports
- Temporary extraction outputs

```

### Extract Images from TES-070

```

### Create New TES-070
1. Run automated test scripts from TestScripts/
2. Capture screenshots and evidence during execution
3. Generate TES-070 document with embedded evidence
4. Save to Generated_TES070s/ folder

## 📝 TES-070 Best Practices

**DO:**
- ✅ Include clear test scenario descriptions
- ✅ Embed screenshots showing actual results
- ✅ Document work unit IDs for traceability
- ✅ Show both expected and actual outcomes
- ✅ Include timestamps and environment details

**DON'T:**
- ❌ Include work unit logs (only UI error messages)
- ❌ Mix sample documents with generated outputs
- ❌ Commit large image files to git (use .gitignore)
- ❌ Store temporary analysis files here (use Temp/ instead)

## 🔄 Workflow

1. **Reference**: Check Sample_Documents/ for format examples
2. **Execute**: Run test scripts and capture evidence
3. **Generate**: Create TES-070 document with embedded screenshots
4. **Save**: Store in Generated_TES070s/ folder
5. **Analyze**: Use tes070_analyzer.py to extract and review evidence
6. **Deliver**: Share final TES-070 documents with stakeholders
